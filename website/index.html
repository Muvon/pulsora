Get Started in 3 Steps

From zero to querying in under a minute

1

Install

cargo install --git https://github.com/muvon/pulsora.git

Or download pre-built binaries from releases

2

Start Server

pulsora
# Server running on http://localhost:8080

Optional: Use custom config with -c pulsora.toml

3

Ingest & Query

# Ingest CSV data
curl -X POST http://localhost:8080/tables/stocks/ingest \
  -H "Content-Type: text/csv" \
  --data-binary @data.csv

# Query time range
curl "http://localhost:8080/tables/stocks/query?start=2024-01-01T10:00:00&limit=100"

Architecture

Proven technologies, optimized for time-series

üóÑÔ∏è RocksDB Storage

LSM-tree architecture optimized for write-heavy workloads. Binary key encoding: table:timestamp:row_id for efficient range scans.

üìä Columnar Blocks

Data stored by column for maximum compression. Type-specific algorithms: Gorilla, delta-of-delta, varint, dictionary encoding.

üîÑ Schema Inference

Automatic type detection from CSV headers. Timestamp column identification. Dynamic schema evolution with validation.

‚ö° Block Caching

Intelligent decompression caching. Configurable cache sizes. LRU eviction for optimal memory usage.

üåê RESTful API

Axum + Tokio async runtime. CORS support. Structured logging. Comprehensive error handling.

üîê ACID Transactions

WriteBatch for atomic operations. Crash recovery. Consistent snapshots for queries.

Simple RESTful API

No query language to learn, just HTTP

Ingest CSV Data

curl -X POST http://localhost:8080/tables/stocks/ingest \
  -H "Content-Type: text/csv" \
  --data-binary @market_data.csv

# Automatic schema inference
# Type detection: Integer, Float, String, Boolean, Timestamp
# Columnar storage with compression

Query Time Range

curl "http://localhost:8080/tables/stocks/query?start=2024-01-01T10:00:00&end=2024-01-01T11:00:00&limit=1000"

# Returns JSON by default
# Sub-millisecond response time
# Efficient block-based scanning

Export to Arrow

curl "http://localhost:8080/tables/stocks/query?limit=10000" \
  -H "Accept: application/vnd.apache.arrow.stream" \
  > output.arrow

# Zero-copy export
# Compatible with pandas, polars, DuckDB
# Efficient columnar format

Get Schema

curl "http://localhost:8080/tables/stocks/schema"

# Returns inferred schema
# Column names and types
# Timestamp column identification

Configuration

TOML-based configuration with sensible defaults

[server]
host = "0.0.0.0"
port = 8080

[storage]
data_dir = "./data"
write_buffer_size_mb = 64    # RocksDB write buffer
max_open_files = 1000         # File descriptor limit

[ingestion]
max_csv_size_mb = 512         # Maximum CSV file size
batch_size = 10000            # Rows per batch

[performance]
compression = "lz4"           # lz4, snappy, zstd
cache_size_mb = 256           # Block cache size

[logging]
level = "info"                # debug, info, warn, error
format = "pretty"             # pretty, json
enable_access_logs = true
enable_performance_logs = true
